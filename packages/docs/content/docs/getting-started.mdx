## Prerequisites

- **Docker** — Docker Engine 24+ with Docker Compose v2
- **Node.js** — 18.x or later (for the CLI)
- At least one AI provider API key (OpenAI or Anthropic)

## Install the CLI

```bash
npm install -g getcito-cli
```

## Initialize Your Instance

Run the interactive setup wizard:

```bash
Getcito init
```

The wizard walks you through:

1. **Config directory** — Where to store your `Getcito.yaml` and `.env` files (default: `./Getcito`)
2. **PostgreSQL** — Run Postgres in Docker (recommended) or provide an existing `DATABASE_URL`
3. **AI Providers** — Configure scrapers (BrightData, Oxylabs, Olostep, DataForSEO) and/or direct model APIs (OpenAI, Anthropic, Mistral, OpenRouter); see [Providers](/docs/user-guide/providers) for details
4. **Start** — Optionally start the stack immediately

Once complete, Getcito generates two files in your config directory:

| File | Purpose |
|------|---------|
| `Getcito.yaml` | Docker Compose configuration for all services |
| `.env` | Environment variables, secrets, and API keys |

## Start Getcito

If you didn't start during init, bring everything up with:

```bash
Getcito compose up -d
```

This pulls the Docker images and starts all services. Once they report healthy, open **http://localhost:1515** in your browser.

## System Architecture

| Service | Role |
|---------|------|
| **Web** | Dashboard UI and REST API |
| **Worker** | Background scheduler that drives prompt runs |
| **Postgres** | Shared database and job queue |
| **LLM APIs** | Direct API calls (OpenAI, Anthropic, OpenRouter) |
| **LLM Scraper** | Scrapers for ChatGPT, Claude, and Google AI Mode (BrightData, Oxylabs, Olostep) |
| **Getcito CLI** | Generates `Getcito.yaml` + `.env` and runs `docker compose` |

## CLI Commands Reference

Once Getcito is running, these commands help you manage it:

```bash
# View live logs
Getcito compose logs -f

# View logs for a specific service
Getcito compose logs -f web

# Check service status
Getcito compose ps

# Stop all services
Getcito compose down
```
